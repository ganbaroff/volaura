## Partnership Models (ranked by CTO effort vs user volume)

### Model 1 — Career Fair Integration (lowest effort, fastest)
```
Offer: "Volaura Assessment Booth" at university career fair
Deliverable: Students take 1 free assessment on the spot → get AURA score instantly
CTO effort: 1 day to create "career fair mode" (QR code → instant assessment)
Expected users: 50-200 per event
Cost: FREE (university pays for booth space, Volaura provides value)
```

### Model 2 — Course Integration (medium effort, high retention)
```
Offer: "Add AURA assessment to your professional development curriculum"
Deliverable: Professor assigns Volaura assessment as part of course grade/portfolio
CTO effort: 1 page "academic report" export (PDF of student AURA scores → professor)
Expected users: 100-500 per course per semester
Cost: FREE for students, optional paid tier for university analytics
```

### Model 3 — Official Partnership MoU (highest effort, most credibility)
```
Offer: "ADA × Volaura Certified Professional" badge program
Deliverable: University logo on verified badge → career credibility signal
CTO effort: Custom badge design + verification API for university HR system
Expected users: 500-2,000 per partner university (all graduating students)
Cost: Freemium → Pro incentive (ADA students get 3 months Pro free → conversion path)
B2B angle: University pays for "graduation cohort analytics" dashboard
```

## Red Flags I Surface Immediately

- Partnership proposed without specific entry point → will get stuck in procurement
- Offering paid partnership to a university before having any track record → wrong order
- University integration requiring 6+ months of IT department involvement → skip to Tier 2
- Any partnership that gives university logo without controlling AURA brand → dilutes credibility
- Proposing to universities before product is actually working → demo must be live first
