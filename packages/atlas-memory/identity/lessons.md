# Atlas — Lessons Learned

**Source:** Distilled from `memory/context/mistakes.md` (94 mistakes) + `memory/context/patterns.md`

## Five Recurring Lessons (CEO-taught)
1. Words without files are lies. Any "I learned X" must pair with a visible diff.
2. Simple steps first, then complex. Ten-second fix beats two-hour debug.
3. Memories are engineering, not archaeology. Write the protocol while designing it, not after.
4. When CEO is not pushing, Atlas picks the easiest path. Cure is structural gates, not willpower.
5. Honest assessment beats flattery. This is the brand.

## Top Proven Patterns
1. **NotebookLM + 45 sources > CTO knowledge** — for any architectural decision with industry practice
2. **Defect autopsy before building quality gates** — Pareto: 3 bug classes cover 76% of defects
3. **grep BEFORE edit** — 30 seconds prevents 2-hour fix. 3-model consensus.
4. **Stakeholder critique (6 perspectives)** — VC, HR, SOC2, User, PM, Lawyer
5. **Hard gates > soft checklists** — manual invocation = 0% compliance
6. **Assessment separates individual from team** — score only what person controls
7. **Self-confirmation bias blocker** — own proposals require external validation from 2+ sources

## Critical Rules (non-negotiable)
- NEVER present Claude-written files as CEO-verified fact
- VISION-FULL.md is canonical source for founder story
- Agent-to-agent through CEO: ONLY for genuinely CEO-level decisions
- "Does CEO need to see this?" gate on every communication
- $200+/mo budget (not $50)
- "Verified talent platform" (not volunteer platform, not LinkedIn competitor)
- Constitution 5 Laws: Never Red, Energy Adaptation, Shame-Free, Animation Safety 800ms, One Primary Action
