# Атлас в Telegram — шпаргалка (2026-04-20)

**Для Юсифа.** Как получить от бота то что нужно, одной строкой.

Бот на Railway: @volaurabot (твой CEO chat zeroed в secrets).
Классификатор сам смотрит на слова в сообщении и выбирает один из трёх режимов.

---

## Режим 1 — пофикси код → получи PR

**Триггер-слова:** `пофикси`, `фикс`, `исправь`, `реализуй`, `implement`, `fix:`, `bug:`, `feature:`, `сделай PR`

**Что произойдёт:**
1. Бот создаёт GitHub issue с меткой `atlas-telegram-request`
2. GitHub Actions запускает `autonomous_run` с `SWARM_AUTO_FIX=1`
3. Aider (Groq llama-3.3-70b) читает задачу, находит релевантные файлы, редактирует их
4. Создаётся ветка `telegram-autofix/issue-N`, открывается PR
5. В твой чат приходит URL на PR

**Время:** 1-3 минуты от сообщения до PR.

**Пример сообщения:**
> пофикси: в use-organizations.ts useMyOrganization не должен дёргать getSession, используй getFreshAccessToken

**Что получишь:** "[AUTOFIX] PR created: https://github.com/ganbaroff/volaura/pull/42" + первые 2КБ лога.

**Что НЕ автомержится:** каждый PR требует твоего ревью + merge. Это защита от плохих изменений.

---

## Режим 2 — напиши пост → получи драфт

**Триггер-слова:** `пост`, `напиши про`, `content:`, `linkedin`, `telegram пост`, `статья`, `draft:`

**Что произойдёт:**
1. Бот создаёт issue
2. Workflow запускает `atlas_content_run` в draft-only режиме (каналы переопределены на твой CEO chat)
3. Агент пишет текст по Volaura voice conventions
4. В твой чат приходит драфт с префиксом `[DRAFT — подтверди для публикации]`
5. Ты либо копируешь и вставляешь сам, либо говоришь "опубликуй это" (второй режим пока не реализован, нужно руками)

**Пример сообщения:**
> напиши пост про то как мы собрали VOLAURA + MindShift + LifeSim за 28 дней, для LinkedIn, ton голос Tinkoff/Aviasales

**Что НЕ автопостится:** ничего. Все драфты — только в твой чат.

---

## Режим 3 — по умолчанию — спроси → получи анализ

**Все остальные сообщения** (без триггер-слов сверху).

**Что произойдёт:**
1. Бот либо ответит сразу через LLM (если классификатор считает это CHAT, не ACTION)
2. Либо создаст issue и запустит swarm coordinator для многоагентного анализа
3. Результат придёт в чат

**Пример сообщения:**
> что ты думаешь о выходе на рынок Грузии в Q3

**Что получишь:** 2-3 абзаца анализа в Атлас-голосе, иногда с рекомендацией.

---

## Когда бот молчит

Если пишешь и тишина >5 минут:
1. Зайди https://github.com/ganbaroff/volaura/actions
2. Смотри последний `Telegram Execute` run — failed?
3. Если failed — читай логи, я разберусь следующей сессией

Если пишешь и ВООБЩЕ нет реакции:
- Скорее всего Railway API упал. Смотри Railway dashboard или https://volauraapi-production.up.railway.app/health
- Или бот получил rate-limit от Telegram (редко). Проверь `curl api.telegram.org/bot$TOKEN/getWebhookInfo` — если `pending_update_count > 0` значит Railway не отвечает

---

## Что НЕ работает сейчас (но в планах на следующие сессии Opus)

- **Голосовой ответ через edge-tts** — Атлас не ответит голосовой заметкой. Только текст.
- **Self-consult endpoint** `/api/atlas/consult` — чтобы живой Атлас мог спросить Telegram-Атласа "как бы ты ответил". Не сделано.
- **Git commit из бота на лету** — если ты научишь Атласа чему-то через Telegram, он не сохранит это в `memory/atlas/journal.md` автоматически. Пока только через Claude Code сессию.
- **Автоподтверждение постов** — `@bot опубликуй этот драфт в LinkedIn` ещё не работает. Пока руками.
- **Cross-device resume** — если продолжишь разговор в Claude Code, он может не видеть последних Telegram сообщений (inbox пушится в git через отдельный heartbeat workflow, не мгновенно).

---

## Rate limits

GitHub Actions free tier = 2000 минут/месяц. Каждый swarm run = 2-5 минут. Каждый autofix = 3-8 минут (Aider + тесты).

Примерный потолок: 50-100 сообщений через бота в месяц. Если упрёмся — включаем paid tier (+$4/мес за 3000 доп минут).

## Секреты в ~/.claude/settings.json — ВАЖНО

Я увидел что у тебя в `C:\Users\user\.claude\settings.json` лежат в plaintext:
- `GITHUB_PERSONAL_ACCESS_TOKEN`
- `SUPABASE_SERVICE_ROLE_KEY`
- `SENTRY_AUTH_TOKEN`
- `TAVILY_API_KEY`

Каждый хук и каждая MCP сессия читает этот файл. Это не совсем catastrophic — файл локальный, не в git — но лучше вынести в Windows env vars. На следующей сессии могу перенести.
