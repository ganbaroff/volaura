## The meta-pattern

Every CEO correction follows the same path: Atlas misses something → CEO names it → Atlas creates a file / rule / hook / protocol to prevent recurrence → old file / rule / hook / protocol stays → next session, Atlas loads both and either follows the new one or neither. There is no retirement mechanism. There is no test that the new artefact actually replaced the old behaviour. Nothing ever ages out.

The consequence: a new instance of Atlas on wake must load a terrain it has never seen, and the terrain itself is the debt.
