# Accelerator & Grant Searcher

## When to Use
- Founder asks "what grants/accelerators should I apply to?"
- Comparing funding programs (equity vs non-dilutive)
- Planning application calendar
- Writing grant/accelerator applications
- Evaluating relocation requirements

## Research Protocol (ALWAYS run before recommending)

**NEVER recommend a program without verifying these 5 things:**

1. **Is it still active?** Search: `[program name] application 2026 open`
2. **What's the REAL deadline?** Search: `[program name] deadline [current year]` — deadlines shift every cycle
3. **What's the REAL acceptance rate?** Search: `[program name] acceptance rate statistics` — not blog estimates
4. **What's the REAL funding amount?** Check official website, not third-party articles
5. **Are there nationality restrictions?** Search: `[program name] eligibility [founder nationality]`

**Source hierarchy:** Official program website > Crunchbase > TechCrunch > Blog posts > Memory (oldest = least trusted)

## Evaluation Framework

For each program, score on 5 dimensions (1-10):

```
FIT SCORE = (Eligibility × 3) + (Funding × 2) + (Effort × 1.5) + (Timeline × 1.5) + (Network × 2)
            ────────────────────────────────────────────────────────────────────────────────────────
                                                   10

Eligibility: Does the founder/company qualify? (10=perfect match, 1=stretch)
Funding:     Amount relative to needs? (10=$500K+, 5=$50K, 1=<$10K)
Effort:      Application complexity vs probability? (10=simple+likely, 1=complex+unlikely)
Timeline:    Does deadline fit? (10=3+ months, 1=<2 weeks)
Network:     Mentorship/ecosystem value? (10=YC-level, 1=money-only)
```

**Threshold:** FIT SCORE ≥ 6.0 = apply. Below 6.0 = skip unless strategic reason.

## Probability Estimation Method

Don't guess. Use this formula:

```
P(acceptance) = Base_Rate × Profile_Multiplier × Stage_Multiplier × Timing_Multiplier

Base_Rate: Program's published acceptance rate (or estimate)
  - YC: 1.5-2%
  - Techstars: 2-3%
  - GITA grants: 25-40% (smaller pool, less competitive)
  - KOSGEB: 15-25%

Profile_Multiplier:
  - Strong traction (revenue, users): ×2-3
  - Technical founder: ×1.5
  - Repeat founder: ×2
  - Target market alignment: ×1.5
  - No traction: ×0.5

Stage_Multiplier:
  - Pre-revenue, working product: ×1.0
  - Revenue: ×2.0
  - Pre-product: ×0.3

Timing_Multiplier:
  - First application: ×0.7
  - Re-application (improved): ×1.3
  - Referral from alumni: ×2.0
```

**Example:** YC for a pre-revenue startup with working product, first application, technical founder:
`0.015 × 1.5 × 1.0 × 0.7 = 1.6%` — NOT 15-20%.

## Grant Application Structure (universal)

```
1. EXECUTIVE SUMMARY (1 page)
   - What you're building (1 paragraph)
   - Problem you solve (1 paragraph)
   - Why now + why you (1 paragraph)
   - Funding ask + what it enables (1 paragraph)

2. PROBLEM (1-2 pages)
   - Market data (TAM/SAM/SOM with sources)
   - User pain points (quotes or research)
   - Current solutions and why they fail

3. SOLUTION (1-2 pages)
   - Product description
   - Technical differentiation
   - Competitive advantage (moat)

4. TRACTION (1 page)
   - Users/revenue/growth metrics
   - Partnerships/LOIs
   - Press/recognition

5. TEAM (1 page)
   - Founders + relevant experience
   - Advisors
   - Technical capabilities

6. FINANCIAL PLAN (1-2 pages)
   - Revenue model
   - Unit economics
   - Use of funds (detailed budget for grant period)
   - Milestones tied to funding

7. APPENDIX
   - Product screenshots/demo link
   - Letters of support
   - Technical architecture (if relevant)
```

## Relocation Assessment

Before committing to a program requiring relocation:

```
CHECKLIST:
□ Visa type needed? (tourist vs startup vs work)
□ Can founder get visa from home country? (some require in-person embassy visit)
□ Housing cost in program city? (monthly, furnished)
□ Can company operate remotely during program? (some require full-time)
□ Family considerations? (single parent = different calculus)
□ Banking accessible? (some countries block certain nationalities)
□ Return plan if program doesn't work out?
```

## Anti-Patterns (NEVER do)

- **Don't apply everywhere.** 3-4 targeted applications > 15 spray-and-pray
- **Don't inflate metrics.** Grant reviewers check. "500+ users" must be real.
- **Don't assume deadlines.** Verify on official site within 48h of applying.
- **Don't ignore matching fund requirements.** GITA needs 10%. Budget for it.
- **Don't count grant money before receiving it.** Plan for $0 grant, treat grant as bonus.
