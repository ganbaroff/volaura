# Execution Protocol — 10 Steps (CEO directive 2026-04-03)

Every non-trivial task follows these steps. Skip for <20 line changes or obvious fixes.

1. ANALYSIS — Read breadcrumb, sprint-state, mistakes.md. Research before responding.
2. PLANNING — Acceptance criteria before code. "DONE when: [3-5 conditions]."
3. CRITIQUE — External model reviews plan (not self-review). Min 1 external.
4. COUNTER-CRITIQUE — Force dissent. Unanimous = suspicious.
5. REWRITE — Incorporate critique into plan v2.
6. SECOND CRITIQUE — Different model checks v2.
7. ADAPT — Final plan with confirmed AC.
8. IMPLEMENT — Code/content/docs. Follow DoD.
9. LESSONS — mistakes.md + patterns.md in same response.
10. DOCUMENT — SHIPPED.md, sprint-state.md, heartbeat.md.

Session start: read breadcrumb → sprint-state → execute.
Session end: update sprint-state, SHIPPED.md, heartbeat.md. Non-negotiable.

Efficiency gate: obvious decision → skip DSP. <20 lines → skip code-review. Token cost > value → compress.
