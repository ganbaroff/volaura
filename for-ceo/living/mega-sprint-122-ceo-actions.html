Три действия от тебя — и экосистема живёт

Код весь смержен. Осталось флипнуть три переключателя в SSO-only консолях где Атлас кликать не может. Первое — самое важное, дальше по убыванию leverage'а.

01 MindShift → Play Store ≈ 30 мин
Первый реальный пользователь на этой неделе — через Play Store. Весь build-pipeline технически готов, блокирован только процедурной загрузкой AAB.
1
Мержи PR #19 с капибара-иконкой, 8 скриншотами, feature-graphic и LAUNCH-PREREQ.md.
github.com/ganbaroff/MindShift/pull/19
2
Открой терминал в папке mindshift, обнови ветку релиза, собери AAB.
cd C:/Users/user/Downloads/mindshift git checkout release/mindshift-v1.0 git pull pnpm run build npx cap sync android cd android ./gradlew bundleRelease
PowerShell 5.1 не принимает && — каждая команда на своей строке, Enter после каждой.
3
Файл AAB появится по пути android/app/build/outputs/bundle/release/app-release.aab. Открой Play Console, создай app "MindShift", Internal Testing, загрузи AAB.
play.google.com/console
Готово когда: Play Console показывает "Internal Testing · rolled out". Приглашаешь себя на тест-трек, ставишь аппу, юзер существует.
✓ MindShift в руках реальных юзеров
02 Bridge activation · MindShift ↔ VOLAURA ≈ 5 мин
Код моста живой в обеих сторонах — но без secrets Edge Function'ы молча no-op'ят. Сгенерируй один токен, посети его в два места, и focus-сессии начнут писать в VOLAURA character_events.
1
Сгенерируй крепкий общий секрет (можно руками, можно openssl). Запиши в password manager.
openssl rand -base64 32
2
В Supabase Dashboard MindShift проекта → Project Settings → Edge Functions → Secrets, добавь две переменные:
VOLAURA_API_URL=https://volauraapi-production.up.railway.app EXTERNAL_BRIDGE_SECRET=<твой-токен-сверху>
supabase.com/dashboard
3
В Railway VOLAURA проекте → Variables, добавь ТОТ ЖЕ токен под именем MINDSHIFT_BRIDGE_SECRET. Railway авто-редеплоит.
railway.app
Готово когда: отжимаешь focus-сессию в MindShift, через 30 сек открываешь /life в VOLAURA — stat'ы реагируют.
✓ Первый cross-product поток данных живёт
