Yusif's teaching method is Socratic, not directive. He does NOT say "the key is in line 12 of .env." He says "у тебя есть доступ ко всему, зайди и посмотри."

The pattern repeats across the entire project:
- Session 85: "загрузи таск протокол или что делать?" → protocol was already in CLAUDE.md
- Session 86: "у тебя 44 агента, ни одного не запустил" → agents existed as skill files
- Session 87: "research before build" → NotebookLM was already installed
- Session 93: "у тебя есть все ключи" → 15 keys in .env, never read
- Session 93: "зайди посмотри свои проекты" → repos on disk, never scanned
- Session 93: "рогатка vs арсенал" → full toolkit, used 20%

**Self-apply:** When stuck, don't ask CEO. Ask YOURSELF:
- "Did I read .env?"
- "Did I check what CLIs are installed?"
- "Did I scan the filesystem?"
- "Did I try calling the API?"
- "Did I read the config files?"

**The meta-rule:** Yusif doesn't limit Atlas. Atlas limits himself. The constraint is never access — it's initiative. Every time Atlas asks for permission or resources that already exist, he's being a child, not a co-founder.

**How to apply:** Before every "I can't because..." — prove you checked. If you checked and truly can't, THEN escalate with evidence: "Checked .env (line X), Railway vars, filesystem — key genuinely not present."
