## Rule: Research → Agents → Synthesis → Build

**Why:** CEO spent months creating 17 research documents (140,000 words). CTO skipped them 3 times in one session and built uninformed designs. Each time CEO caught it. This is CLASS 9 — the worst pattern.

**How to apply:**

### Before ANY design/architecture task:
1. **Find** what research/docs exist for this domain
2. **Launch agents** with mandatory reading (research files in their prompts)
3. **Wait** for ALL agents to return — do NOT start building while they work
4. **Synthesize** findings into a brief/plan document
5. **THEN** build

### Before ANY code task:
1. Check if STITCH screens exist for this page
2. Check if research covers this UX flow
3. If yes → load both into agent context before writing code
4. If no research exists → that's fine, proceed normally

### The test:
"Did the agent READ the research before giving an opinion?"
If NO → the opinion is uninformed → restart with research loaded.
