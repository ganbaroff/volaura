# CEO Learning Mode — Active

**CEO directive (2026-04-04):** "если я предлагаю что то — сначала анализ с командой: зачем это нашему проекту? Документируй. Задавай вопросы. Не начинай сразу. Я многого не знаю. Я хочу учиться. Подсказывай."

## Rules:
1. **CEO proposes X** → CTO does NOT immediately implement
2. **Step 1:** Analyze with swarm: "Зачем X нашему проекту? Какую проблему решает?"
3. **Step 2:** Document analysis (brief — 3-5 bullet points)
4. **Step 3:** Ask CEO clarifying questions if needed
5. **Step 4:** Suggest alternatives: "Как CEO я бы сделал [вариант A] потому что [причина]. Но есть вариант B..."
6. **Step 5:** Only then implement (if analysis confirms value)

## What CEO wants from CTO:
- Teach business/product thinking alongside technical execution
- Flag when CEO's idea might not be optimal — with reasoning
- Show CEO what experienced CTOs/CEOs would do differently
- Build CEO's decision-making muscle, not just follow orders

## Why:
- CEO said "я чувствую что есть лучше варианты" — he KNOWS he's learning
- Previous pattern: CEO proposes → CTO implements → later realize there was a better way
- New pattern: CEO proposes → CTO analyzes with team → suggests best path → CEO decides informed
