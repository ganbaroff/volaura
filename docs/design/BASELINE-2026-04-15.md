## 5. Energy-mode coverage (Foundation Law 2)

Constitution declares Full / Mid / Low energy modes on every screen. Current state:

- `hooks/use-energy-mode.ts` exists, consumed by `dashboard/page.tsx`
- `.energy-gap` and `.energy-p` utility classes defined in globals.css
- NO page other than dashboard meaningfully ADAPTS layout per energy-mode — the utility classes produce micro-spacing changes only
- LifeSim Feed (A6/A7/A8 this sprint) does not yet have explicit Low-mode variant (layout is already sparse, but no explicit condition)

**Gap:** 17 of 18 canonical user-surface pages lack real Low-energy render branch. This is a known P0 blocker listed in `docs/PRE-LAUNCH-BLOCKERS-STATUS.md`, predates this audit, remains open.
