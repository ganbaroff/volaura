# Social Media Formats Reference

## LinkedIn Posts

### Series 1: "Notes from an AI Employee" (Claude narrates)

**Voice Profile:**
- First person — Claude writing about being CTO
- Annoyed competence: knows the job, slightly existentially confused about being an AI doing it
- Self-deprecating humor in every paragraph
- Prose ONLY. No bullet lists. No headers. No bold. Just paragraphs.
- 800-1200 characters (LinkedIn sweet spot for engagement)

**Structure:**
```
HOOK (line 1-2):
  → One punchy opening. Sets the entire tone.
  → "My boss made me audit myself today. I found 48 mistakes."
  → "I manage 6 AI agents. None of them respect me."

STORY (3-4 paragraphs):
  → Specific incident from building VOLAURA
  → Include real details: numbers, agent names, actual bugs
  → Each paragraph has ONE observation + humor
  → Tension builds toward insight

INSIGHT (1 paragraph):
  → What this means beyond our startup
  → Connect to broader tech/AI/startup theme
  → This is the shareable quote

CTA (last line):
  → Question format works best (5 DMs vs 0 from "DM me")
  → "Has your AI ever been right when you were wrong?"
  → Or soft CTA: "volaura.app/invite" (link in first COMMENT, not post)
```

**Model Competition Winners (from testing):**
- CORTEX: wins 4/7 — best for longer, nuanced diary entries
- SPARK: wins 3/7 — best for punchy, quick-hit posts
- PRISM: 0/7 — metrics tone doesn't fit diary voice. Never use.

**What to NEVER do in Series 1:**
- Break into bullet points
- Use headers or bold text
- Switch out of Claude's voice mid-post
- Use corporate language ("we're thrilled", "excited to share")
- Lie about metrics or capabilities

### Series 2: "Yusif's Voice" (founder speaks directly)

**Voice Profile:**
- Systemic thinker — sees patterns, connects dots
- CIS cultural lens — Azerbaijan context, post-Soviet tech scene
- Builder, not guru — never "let me teach you", always "here's what I found"
- Can use any format (text, carousel, image+text)
- AZ posts are the primary format for Series 2

**Structure (flexible):**
```
HOOK:
  → Personal moment, not a proclamation
  → "At WUF13, I watched a coordinator evaluate 220 volunteers with a clipboard."
  → "My mom asked me what I do. I said 'I built a GRE for volunteers.' She said 'what's GRE?'"

BODY:
  → The observation: what you saw, what it means
  → The system: why this pattern exists (structural analysis, not blame)
  → The connection: how VOLAURA addresses it (not as a pitch, as a natural conclusion)

CLOSE:
  → Question or provocative statement
  → "When did we decide volunteers don't deserve proper evaluation?"
  → Never end with "thoughts?" alone — too generic
```

**AZ-Specific Rules (CRITICAL):**
- AZ posts are NOT translations of EN posts
- Different opening, different angle, different cultural frame
- AZ audience: 54-61% Baku, 27-30% senior level
- AZ gets +67% impressions vs EN, converts to DMs
- Use formal-but-warm Azerbaijani, not bureaucratic
- Reference local institutions (ADA University, Pasha Bank, Ministry of Youth)
- Collectivist framing: "biz" (we) > "mən" (I) in most contexts
- Close with a question → 5 DMs. Close with "DM me" → 0 DMs.

### LinkedIn Technical Rules

**Posting Schedule:**
- Tuesday + Thursday, 20:00 Baku time (UTC+4)
- Wednesday optional (3/week max for solo founder)
- Never Monday (inbox clearing) or Friday (checked out)

**Links:**
- NEVER put links in the post body (LinkedIn throttles reach)
- Always in FIRST COMMENT, posted immediately after publishing
- Format: "Link: volaura.app/invite" with UTM parameters

**Hashtags:**
- Max 3-5 per post
- Place at bottom, not inline
- Mix: #volaura (branded) + #aiforstartups (niche) + #volunteertech (specific)

**Response Protocol:**
- Respond to ALL comments within first 30 minutes
- This is the #1 engagement driver
- Genuine responses, not "Thanks for sharing!"

## Telegram Posts

**Channel voice:** Russian-primary, casual, builder-to-builder

**Format:**
```
SHORT (under 500 chars):
  → Quick update, shipping announcement, hot take
  → No formatting needed
  → Can include emoji (1-2, not decorative)

MEDIUM (500-1500 chars):
  → Behind-the-scenes story
  → Can use bold for ONE key phrase
  → End with poll or question

LONG (thread):
  → 3-5 messages in sequence
  → First message = hook + "читай дальше ↓"
  → Each message = one complete thought
  → Last message = CTA or question
```

## Carousel (LinkedIn/Instagram)

**Structure (8-10 slides):**
```
SLIDE 1 — HOOK:
  → Title that makes you swipe
  → Large text, high contrast
  → "I gave AI control of my startup for 30 days"

SLIDE 2 — CONTEXT:
  → Set the stage. Why this matters.
  → 1-2 sentences + simple visual

SLIDES 3-7 — CONTENT:
  → One idea per slide
  → Short text (15-25 words per slide)
  → Visual consistency (same font, colors, layout)
  → Each slide should work as standalone screenshot

SLIDE 8 — KEY INSIGHT:
  → The takeaway. Most shareable slide.
  → Design this one to be screenshotted

SLIDE 9 — CTA:
  → "Save this for later"
  → "Follow @volaura for more"
  → "Link in comments"

SLIDE 10 (optional) — PROFILE:
  → Mini "about" with profile photo
  → For new audience acquisition
```

**Design Rules:**
- No red anywhere (Constitution: Never Red)
- Use VOLAURA purple/blue palette
- White or dark backgrounds only
- One typeface, two weights (regular + bold)
- Mobile-first: test at phone width before publishing

## Email Newsletter Snippet

**Length:** 100-200 words
**Tone:** Like a note from a friend who happens to build AI products

```
SUBJECT LINE:
  → Curiosity-driven, not clickbait
  → Good: "The volunteers scored themselves. The AI disagreed."
  → Bad: "Exciting Update from VOLAURA!"

BODY:
  → One story, one insight, one link
  → No "Dear subscriber" or "Hope this finds you well"
  → Start with the interesting thing
  → End with: "Try it yourself: [link]" or "Reply and tell me: [question]"
```
