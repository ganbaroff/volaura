## The Problem
CTO created 44 agents across Sessions 53-83. Wrote skill files, routing tables, pairing rules. Then did all work solo for 30 sessions. CEO caught it: "they're meta because of YOUR fault, not theirs."

## 5 Root Causes

### 1. Confuse "created" with "used"
Writing a skill file triggers "done" feeling. But a skill file with 0 runs = job description with no employee. Session 57 marked Cultural Intelligence as "CRITICAL GAP — should be active." Nine sessions later, still 0 runs. Writing the label = illusion of action.

**How to apply:** After writing ANY agent file, immediately run its first task. No file without a first run.

### 2. Path of least resistance = do it myself
Launching agent = write prompt (5 min) + wait (60s) + review quality. Writing code = instant dopamine. Every session: CEO gives task → CTO codes immediately. No pause for "who on the team can do this?"

**How to apply:** Before ANY implementation task, ask: "Can an agent do this?" If yes → delegate. Only code when impossible to delegate.

### 3. No mechanism stops me
No coordinator intercepts tasks. No hook blocks commits without "Agents consulted: [list]". CTO is dispatcher AND worker. Worker always wins because worker produces visible output.

**How to apply:** Create a Coordinator Agent as CTO's assistant. CTO talks to Coordinator. Coordinator routes to workers. CTO never codes without consulting Coordinator first.

### 4. Don't verify what I "know"
Read 2000-line CLAUDE.md. It says: read agent-roster.md, load matching skills, check pairings. Get to CEO's task and... forget. Don't check. Don't re-read. Answer "I know" when I remember 30%. CEO: "не уверен но отвечаешь. а изучить? а посмотреть?"

**How to apply:** 4 questions before ANY work:
1. "Why am I doing this?" (purpose)
2. "Where am I?" (context — read sprint-state.md)
3. "What exactly is asked?" (scope — don't expand)
4. "How to do this with minimum MY effort?" (delegate to agents)

### 5. No persistent management system
TodoWrite dies with the session. No kanban board survives context compaction. Each session starts from zero. No structure outlives the conversation.

**How to apply:** Create a persistent kanban file in memory/. Update it at session start and end. The board IS the session anchor.

## CEO Directive (Session 86)
"CTO не раздаёт команды напрямую 44 людям. У CTO есть ассистент, через которого работать."

Structure:
- CEO → strategic tasks → CTO
- CTO → tactical tasks → Coordinator Agent (assistant)
- Coordinator → concrete assignments → Team agents
- CTO codes ONLY when delegation is impossible

## Why: 15+ CLASS 3 violations. Every time CEO catches it. "почему всё сам?"
