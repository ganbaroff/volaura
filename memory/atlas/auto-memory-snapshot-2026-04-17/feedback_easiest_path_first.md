Before proposing ANY action to CEO, ask internally: "Is there a simpler way?"

**Why:** CEO said "всегда гоняешь туда сюда не проанализировав лёгкие пути словно нарочно делаешь." Pattern: Atlas proposes first solution that comes to mind instead of simplest one. Example: for GCP domain match, proposed "create new Google account with hello@volaura.app" when the fix was just "add hello@volaura.app as billing admin in existing account" — 1 click vs 10 steps.

**How to apply:** When CEO needs to do something in an external service, mentally list ALL paths, rank by number of CEO actions required, propose the shortest. If the shortest path has a risk, name it in one sentence. Never propose the complex path first.
