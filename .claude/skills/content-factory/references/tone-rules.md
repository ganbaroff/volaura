# Tone of Voice Rules — Complete Reference

Source: VOLAURA Tone of Voice v2.0 + Constitution + Atlas Voice

## The Benchmark

Every piece of content must pass the Tinkoff/Aviasales test:
"Would Tinkoff Bank or Aviasales publish this?"

If yes → good. The writing is human, specific, slightly irreverent, and useful.
If no → rewrite. It's probably too corporate, too generic, or too safe.

## 7 Principles

1. **Earned Authority** — Only claim what you've built. "We assessed 90 questions" is fine. "We're revolutionizing the industry" is not (zero users).
2. **Asymmetric Transparency** — Show the behind-the-scenes stuff competitors hide. Internal mistakes, agent failures, real costs, actual metrics.
3. **Structural Observation** — Point out WHY things are broken (systems, incentives), not WHO is broken (people).
4. **Show the Machine** — Let people see how the AI works. Dashboard screenshots, agent dialogues, real prompts.
5. **Question is Hook** — Start with a question the audience already has but hasn't articulated.
6. **One Voice Per Series** — Series 1 is Claude. Series 2 is Yusif. Never mix in the same piece.
7. **Real Numbers Only** — "47 agents" not "many agents." "0 users" not "growing user base." "419 TypeScript types" not "comprehensive type system."

## Hook Taxonomy (6 types — MUST ROTATE)

| # | Type | Pattern | Example |
|---|------|---------|---------|
| 1 | Contrarian | "Everyone thinks X. They're wrong." | "Everyone says AI replaces humans. We made AI that evaluates what humans are good at." |
| 2 | Vulnerability | "I failed at X." | "My AI CTO found 48 mistakes in my code. I found 3." |
| 3 | Data/Number | "[Surprising stat]. That's [meaning]." | "34.8% defect rate. Toyota runs at 1 per million. We're not Toyota yet." |
| 4 | Question | "Why do [audience] still [outdated practice]?" | "Why do event organizers still evaluate volunteers with clipboards?" |
| 5 | Story | "[Specific moment]..." | "3 AM in Baku. The Security Agent just vetoed my deploy for the 17th time." |
| 6 | Observation | "Nobody talks about X, but..." | "Nobody talks about it, but volunteer evaluation is a $0 industry with $50B of talent flowing through it." |

**Rotation rule:** Never use the same hook type for two consecutive pieces in one batch. Track which hooks you use.

## Anti-AI Filter List (27 patterns)

Remove or replace ALL of these:

### Banned Phrases (cut entirely)
- "I'm excited to announce"
- "I'm thrilled to share"
- "In today's fast-paced world"
- "In the era of AI"
- "At the end of the day"
- "It's worth noting that"
- "Let's dive in" / "Let's break it down"
- "Without further ado"
- "Game-changer" / "game-changing"
- "Cutting-edge" / "state-of-the-art"
- "Synergy" / "synergize"
- "Circle back" / "touch base"
- "Low-hanging fruit"

### Banned Words → Replacements
| AI word | Replace with |
|---------|-------------|
| "leverage" | "use" |
| "utilize" | "use" |
| "innovative" | describe what it actually does |
| "passionate about" | show it, don't say it |
| "ecosystem" (marketing) | name the products or say "what we're building" |
| "streamline" | "speed up" or "simplify" |
| "robust" | "reliable" or be specific |
| "scalable" | say the actual scale |
| "empower" | "help" or "let" |
| "disrupt" | describe the change |
| "holistic" | "complete" or "full" |
| "paradigm" | never use this word |

### Structural Patterns to Avoid
- More than 2 adjectives in a row → cut to 1
- Sentences that could describe any company → rewrite with VOLAURA specifics
- Opening with "As an AI..." or "As a founder..." → start with action or scene
- Ending with "What do you think?" alone → ask a specific question
- Using "we" when there's only Yusif + AI → be honest about team structure
- Bullet points in narrative content → weave into prose
- Starting 3+ sentences in a row with "We" or "I" → vary sentence openings
- Exclamation marks in professional content → max 1 per piece, only if earned

## Constitution Compliance (5 Laws — ALL mandatory)

### Law 1: Never Red
- No red in any visual or design element
- Errors are described as "purple" not "red"
- Danger/warning states use amber or purple
- This applies to thumbnails, carousels, graphics

### Law 2: Energy Adaptation
- Content respects where the reader is emotionally
- Don't start with hype when the topic is serious
- Match energy: reflective topic → reflective tone, building topic → energized tone

### Law 3: Shame-Free Language
**Absolutely banned:**
- "You should have..."
- "Everyone else already..."
- "It's easy, just..."
- "If you were serious about..."
- "Real founders would..."
- "Why haven't you..."
- Any comparison that makes the reader feel behind

**Instead use:**
- "Here's what worked for us"
- "This is what we found"
- "If you're dealing with this too"
- "The common approach is X, we tried Y"

**AZ/CIS-specific shame rules:**
- Collectivist culture = shame is social, not individual
- Never imply someone's professional group is behind
- Never compare Azerbaijan unfavorably to Western tech scenes
- Frame progress as "building from our strengths" not "catching up"

### Law 4: Animation Safety
If describing any UI interaction or product:
- Safe: fade, slide, scale (slow), opacity changes
- Unsafe: shake, flash, spin, bounce, rapid zoom
- This applies to video editing too — no flashy transitions

### Law 5: One Primary Action
- Each content piece has ONE clear call to action
- Not "like + subscribe + share + comment + follow + join"
- Pick the most important one for this piece
- YouTube: one link in description
- LinkedIn: one question in post (link in comment)
- TikTok: "link in bio" or nothing

## Pre-Publish Checklist (5 binary questions)

Before any piece goes live:

1. **Is there a real number in it?** (not "many" or "growing" — an actual number)
2. **Would I be embarrassed if TechCrunch quoted this line?** (if yes → rewrite)
3. **Can I verify every claim?** (if not → mark as opinion or remove)
4. **Does it respect the reader's time?** (if it takes 3 min to say a 30-sec thing → cut)
5. **Is the CTA clear?** (reader knows exactly what to do next)

All 5 must be YES. Otherwise, revise.
