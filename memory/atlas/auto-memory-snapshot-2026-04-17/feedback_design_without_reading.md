## Нарушение

Session 90: создал 8 Figma экранов (Expert Verification, Org Dashboard, Life Simulator, BrandedBy и др.) и 88 Figma Variables не прочитав:
- бэкенд роуты для этих фич (существуют ли вообще)
- character_events schema
- IRT engine — реальный или заглушка
- BrandedBy, Life Simulator код
- crystal economy implementation
- реальный user data из Supabase
- sprint history что уже сделано

**Критичность:** дизайн оторван от реальности. Экраны для фич которые могут не существовать в бэкенде.

**Why:** Что случилось несмотря на 4 проверки "уверен ли ты?"

Каждый раз когда CEO спрашивал "готов?" — я проверял ТАКТИКУ (правильный ли инструмент, корректен ли формат). Не проверял ОСНОВАНИЕ (знаю ли я вообще систему которую проектирую).

Вопрос "уверен что это лучший путь?" я интерпретировал как "правильный ли Code Connect?" Правильная интерпретация: "знаешь ли ты продукт достаточно чтобы это делать?"

Это scope mismatch: стратегический вопрос воспринят как тактический.

**How to apply:**

**ПЕРЕД любой дизайн/UI/архитектурной работой — обязательная проверка:**

```
□ Я читал бэкенд роуты для каждого экрана который собираюсь рисовать?
□ Я знаю что character_events содержит и кто туда пишет?
□ Я видел production данные (реальные пользователи, completion rate)?
□ Для каждого продукта в экосистеме — я читал его код или только docs/memory?
□ Если рисую интеграцию — я видел реальный data flow в коде?
```

Если хотя бы один ответ "нет" → **СТОП. Читать сначала.**

**Правило:** Рисуй экраны только для фич бэкенд которых ты читал.

**Конкретное исправление для этой ошибки:**
1. Дождаться результата ecosystem-audit агента
2. Сравнить с нарисованными экранами
3. Удалить или пересмотреть экраны для несуществующих фич
4. Пересоздать Variables только для токенов которые реально используются в коде
