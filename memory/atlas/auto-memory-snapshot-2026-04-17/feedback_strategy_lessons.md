# Lesson: Comprehensive ≠ Correct

## What Triggered This

Session 44+ BrandedBy analysis. Produced a 600-line plan covering all dimensions.
Yusif challenged: "is this the BEST plan?"
Self-critique revealed 7 critical weaknesses in an apparently comprehensive deliverable.

## The Pattern of Mistakes

### Mistake Class A: Missing Unit Economics
**Pattern:** Writing pricing/revenue strategies without modeling COGS.
**Example:** "Sign 10 celebrities" without modeling what celebrities cost vs revenue.
**Rule:** NEVER recommend a monetization model without: COGS per unit, gross margin %, break-even volume.

### Mistake Class B: Chicken-and-Egg Blindness
**Pattern:** Assuming Step 3 is possible without noticing it requires Step 3 to already exist.
**Example:** Self-serve funnel needs demo videos → demo videos need signed celebrities → need product to work.
**Rule:** For any new product, explicitly map: what does the customer need to see to trust? What do I need to have built to show them that? What do I need first to build THAT?

### Mistake Class C: Timeline Without Resource Model
**Pattern:** Estimating timelines for a 5-person team when a solo founder + Claude is executing.
**Rule:** Before any timeline, ask: Who is executing? How many hours/week? What is parallel load?
**BrandedBy specific:** Yusif runs Volaura in parallel. BrandedBy = 15-20 hours/week maximum.

### Mistake Class D: Aspirational Expansion vs Operational Expansion
**Pattern:** Writing "open Tbilisi office" in Month 15 without addressing: legal entity, bank account, hire, employment law.
**Rule:** Expansion steps must specify: legal entity requirement, estimated setup time, local hire needed, first customer needed before entry.

### Mistake Class E: Missing the Critical First Question
**Pattern:** Building a celebrity licensing strategy without knowing if ANY contracts exist.
**Rule:** For any strategy involving a key asset (celebrity contracts, patents, existing customers), ask about current status FIRST. Build strategy from reality, not from zero.

### Mistake Class F: Technical Correctness ≠ Business Correctness
**Pattern:** Gave correct Stripe API implementation but wrong commercial recommendation (public pricing before case studies).
**Rule:** Technical implementation questions are subordinate to commercial sequencing questions. Ask: is the buyer ready to buy this yet? Before: how do I process the payment?

## What World-Class Strategy Looks Like

1. Starts with: "What do we have TODAY?" not "Here's what you should build"
2. Models unit economics before recommending pricing
3. Identifies the chicken-and-egg and solves it explicitly
4. Timeline = resource-constrained, not theoretical
5. Expansion = operational checklist, not growth slide
6. Raises the uncomfortable question the founder hasn't asked yet
7. The plan is shorter and more actionable, not longer and more comprehensive

## The Calibration Question (Run This Before Finalizing Any Strategy)
"If I shipped this plan to a Series A investor who has seen 500 startups,
what are the 3 things they'd immediately poke holes in?"
Then fix those 3 things before calling it done.
