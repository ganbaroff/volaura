# Swarm Agent Cards

CEO directive 2026-04-21: "пусть они настраивают самих себя для начала так как они хотят."

Each model in the Atlas swarm wrote its own agent card — unsolicited, verbatim, in its own voice.
No dictation. No expected format enforced. What you read is what each model chose to say.

---

## What is this directory

`self-definitions/` holds one `.md` file per model. Each card was generated by running
`scripts/swarm_self_setup.py`, which sends the same open meta-prompt to every model and saves
the response verbatim with a timestamp header.

The cards are self-assessments, not specs. They tell you:
- What the model thinks it is good at
- What it explicitly refuses (important signal)
- What prompt format gets the best out of it
- How it sees its role relative to teammates

## When to update a card

- New model added to arsenal (`memory/atlas/arsenal.md`) — run setup for that model and add to this README.
- CEO corrects a model's behavior pattern — re-run setup and note the correction in the card manually.
- Model version bump (e.g. deepseek-chat → deepseek-chat-v3.5) — re-run setup, new timestamp overwrites.
- Card shows identity confusion (model called itself by wrong name) — re-run and note in `memory/atlas/incidents.md`.

To refresh all cards: `python3 scripts/swarm_self_setup.py` from repo root.
To refresh one model: edit `make_models()` to run only that slug, or add `--slug` filter (future improvement).

## How to use cards when spawning agents

1. Before assigning a task to a model, read its card — especially sections 3 (tasks for me) and 4 (tasks NOT for me).
2. If your task falls in section 4 of your first-choice model, pick another.
3. Quote the card's preferred prompt format in your spawn prompt. Example: "You said you prefer structured JSON output requests — here is one."
4. If the card says the model excels at X and you need X — say so explicitly. "Your card says you excel at multi-step reasoning. This task requires that."
5. After a spawn, if the output quality contradicts the card — note it in `memory/atlas/lessons.md` and flag for card refresh.

The cards are living documents. A model that writes a bad card reveals something real: either its self-knowledge is poor (use cautiously) or its response quality at the time was off (retry).
