03 Three confirmed gaps

These are the objections that survived counter-evidence. Each changes the plan.

T47-2 · shame condition
Foundational users land in shame-muted pill
apps/web/src/components/dashboard/aura-score-widget.tsx:22
none: { pill: "bg-muted text-muted-foreground border-border" }

Users who complete ≥3 competencies but stay score <40 land in muted shame treatment. Violates Constitution Law 3 (shame-free). Fix: recolor none to neutral indigo (#6366f1, radar-chart convention), contextual label "Assessing"/"Foundational", strength-first caption.

T48-2 · withdrawal path missing
Art 7(3) right to withdraw not honoured
Grep pattern withdrawn_at|withdrawal|revoke across packages/ecosystem-compliance/ → 0 hits.

consent_events is append-only (correct for audit trail), but no withdrawal model exists. Fix: add event_type='withdrawn' row; decision-time latest-event lookup; "Withdraw AI consent" CTA in /settings/decisions; assessment.py reads latest state before writing decision log.

T48-4 · art 22(3) loop open
human_review_requests has zero consumer
Grep across ecosystem → no readers, no operator UI. Art 22(3) requires human-intervention capability — not just a queue row.

User CTA currently writes to a dead queue. Fix: minimal /atlas/human-reviews admin route (flag-gated, admin-only), CEO reviews manually, outcome emailed via existing infra.

08 Root-cause fix proposed

Symptom: §14.6 ticket estimates written before Strange v2 ran.

Pathway: bias toward visible artifact ("here's the plan") faster than hidden gate ("validate first"). Plan-before-validation felt productive. It was premature.

Structural fix — append to .claude/rules/atlas-operating-principles.md:

If response contains "plan" or "ticket list" or "estimate" for multi-file changes AND no Gate 1 external-model call has fired in this session — abort draft, run Strange v2, then write the plan.
